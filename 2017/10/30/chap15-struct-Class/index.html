CHAPTER15 / Structure & Class

구조체와 클래스

구조체와 클래스는 서로 닮기도 했고 고유 특징도 가지고 있다.
어떤 상황에서 둘 중 어느 것을 사용하느냐를 판단해야 한다.


구조체(Struc)

  • 구조체는 서로 연관된 데이터들을 한데 묶은 타입이다.
  • 데이터를 어떤 하나의 타입에 두고 싶을 때 사용한다.(하나의 타입으로 캡슐화 됨)
  • 구조체와 열거형은 값 타입이기 때문에 인스턴스의 프로퍼티 값을 변경할 메서드에는 mutating 키워드가 붙어야 한다.


CODE


클래스(Class)

  • 참조타입으로 서로 관련된 데이터를 구체화하는 데 사용된다.

  • 구조체와는 다르게 상속 으로 인해 매우 일반적인 관점 에서 정의된다.

    • 상속(inheritance) : 한 클래스가 다른 클래스 안에서 정의되는 관계.
    • 상속으로 정의된 클래스는 서브클래스 라고 부르며, 이 서브클래스를 포함한 클래스를 슈퍼클래스 라고 부른다.
    • 서브클래스는 슈퍼클래스의 프로퍼티와 메서드 를 상속한다.

  • 오버라이드 (override) : 슈퍼클래스에 정의된 메서드를 다시 정의한다는 뜻이다.

    • 오버라이드 막기 : 오버라이드를 금지해야 할 경우에 final 키워드나 static 키워드를 사용한다.

class newBaby: getBaby { //getBaby 클래스를 상속
    //name 프로퍼티를 추가
    var name = "hyunah"

    //born 함수를 override
    override func born(Color: String) {
        print("\(Color)색 눈동자 입니다. ")

        //부모class의 born 메소드를 의미한다
        super.born(Color: "yellow")
        }
    }

var hyun = newBaby()
hyun.born(Color: "pink")  

//출력 : pink색 눈동자 입니다.
//ellow 눈동자인 1살 애기가 태어났습니다.
